# Supply Chain Security

All Aotanami release artifacts are signed and attested. This guide explains how to verify them.

## Verifying Container Images

Every release image is signed with [Cosign](https://github.com/sigstore/cosign) using keyless signing (Fulcio + Rekor transparency log).

```bash
# Verify the image signature
cosign verify ghcr.io/zelyo-ai/aotanami:v0.1.0 \
  --certificate-identity-regexp='https://github.com/zelyo-ai/aotanami' \
  --certificate-oidc-issuer='https://token.actions.githubusercontent.com'
```

## Verifying Helm Charts

Helm OCI charts are also Cosign-signed:

```bash
cosign verify ghcr.io/zelyo-ai/charts/aotanami:0.1.0 \
  --certificate-identity-regexp='https://github.com/zelyo-ai/aotanami' \
  --certificate-oidc-issuer='https://token.actions.githubusercontent.com'
```

## SBOMs

Each release includes Software Bills of Materials in both SPDX and CycloneDX formats, generated by [Syft](https://github.com/anchore/syft).

SBOMs are:
1. Attached to the container image as OCI artifacts
2. Uploaded as GitHub Release assets

### Download SBOM from Release

```bash
# Available on the GitHub release page as:
# - sbom-spdx.json
# - sbom-cyclonedx.json
```

### Extract SBOM from OCI

```bash
cosign download sbom ghcr.io/zelyo-ai/aotanami:v0.1.0
```

## SLSA Provenance

Release images include [SLSA Level 3](https://slsa.dev/) provenance attestations, certifying the build process integrity.

```bash
# Verify SLSA provenance
slsa-verifier verify-image ghcr.io/zelyo-ai/aotanami:v0.1.0 \
  --source-uri github.com/zelyo-ai/aotanami
```
